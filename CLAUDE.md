## Markdown Formatting Rules

When editing any `.md` file in this repo:
- Use one sentence per line (semantic line breaks).
- Never wrap mid-sentence at a character limit.
- A sentence ends with `.`, `?`, or `!` followed by a newline.
- Blank lines between paragraphs only — not between list items
  unless they are multi-sentence blocks.
- Do not reflow or reformat existing lines unless explicitly instructed.
